# AI Agent Context — Android Application Installer

> **Purpose:** Give an AI agent everything it needs to work on this project without reading through every source file. This document covers project structure, conventions, key patterns, scripts, and workflows.

---

## Project Overview

A **Tauri 2** desktop app that installs `.apk` and `.aab` files onto connected Android devices. The app downloads and manages its own Android toolchain (ADB, bundletool, Java JRE) — users don't need the Android SDK.

| Layer | Technology |
|-------|-----------|
| Framework | Tauri 2 |
| Backend | Rust |
| Frontend | React 19 + TypeScript + Vite |
| Icons | Lucide React |
| Testing | Vitest + React Testing Library (frontend), `cargo test` (Rust) |
| CI/CD | GitHub Actions |

**Repository:** `havokentity/android-application-installer`
**Current version:** Check `package.json` → `"version"` (canonical source of truth).

## Commit Message Conventions

Commits use **prefix-based** conventional commits. The changelog and release scripts auto-categorize by prefix:

| Prefix | Changelog Section |
|--------|------------------|
| `add:` / `feat:` / `feature:` | ### Added |
| `fix:` / `bugfix:` | ### Fixed |
| `update:` / `refactor:` / `chore:` | ### Changed |
| `delete:` / `remove:` | ### Removed |
| `release:` | Skipped (release commits) |
| _(anything else)_ | ### Other |

**Multi-category commits:** If the commit body contains lines starting with different prefixes, each line is categorized independently.

Example:
```
add: stop app button
fix: layout shift bug
update: header styling
```

## CHANGES.md Format

Follows [Keep a Changelog](https://keepachangelog.com/). Structure:

```markdown
# Changelog

All notable changes to Android Application Installer are documented here.

The format follows [Keep a Changelog](https://keepachangelog.com/).

---

## [Unreleased]

(new entries go here, auto-populated by `npm run changelog`)

---

## [1.8.4] — 2026-04-11
### Fixed
- **Bold summary** — detailed explanation

---

## [1.8.3] — 2026-04-11
...
```

**Key rules:**
- New entries go under `## [Unreleased]`
- The release script promotes `[Unreleased]` → `[x.y.z] — YYYY-MM-DD` automatically
- Each version section is separated by `---`
- Entries use `### Added`, `### Fixed`, `### Changed`, `### Removed` sub-headings
- Each entry starts with `- **Bold summary** — description`

## Release Process (Summary)

> See `docs/release-guide.md` for the full step-by-step guide.

1. If on a feature branch → merge to `main` and push
2. Run `npm run changelog` to populate `[Unreleased]` from commits
3. Review/edit `CHANGES.md`, commit if edited manually
4. Run `npm run release:patch` (or `:minor` / `:major`)
5. The script: runs tests → bumps version → promotes changelog → commits → tags → pushes
6. CI builds all platforms → creates a GitHub Release draft
7. Review the draft on GitHub → publish when ready (`npm run release:publish`)

## Architecture Quick Reference

### Frontend State Management

State is distributed across **9 custom hooks** in `src/hooks/`. `App.tsx` orchestrates them and wires into the component tree. No external state library.

| Hook | Domain |
|------|--------|
| `useToolsState` | Tool download status, progress, staleness |
| `useDeviceState` | Device list, selected device, tracking, device details |
| `useFileState` | Selected file(s), type, drag-drop, batch files, package name |
| `useAabSettings` | Java/bundletool paths, keystore config, signing profiles |
| `useUpdater` | Auto-updater check, download progress, preferences |
| `useLayout` | Portrait/landscape, theme, panel width |
| `useWirelessAdb` | WiFi pairing, connection, mDNS discovery |
| `useKeyboardShortcuts` | Global keyboard shortcut bindings |
| `useEasterEgg` | Easter egg state |

### Typed IPC Layer

All Tauri `invoke()` calls go through `src/api.ts` — fully typed functions, zero string-based command names in components. When adding a new Tauri command:
1. Add the `#[tauri::command]` function in Rust
2. Register it in `lib.rs`
3. Add a typed wrapper in `api.ts`
4. Call the wrapper from hooks/components

### Rust Backend Modules

| Module | Responsibility |
|--------|---------------|
| `lib.rs` | Command registry (thin entry point) |
| `adb.rs` | ADB operations (install, launch, stop, uninstall, devices, wireless) |
| `cmd.rs` | Command execution, cancellation tokens, `CREATE_NO_WINDOW` on Windows |
| `java.rs` | Java & bundletool detection, key alias listing |
| `package.rs` | Package name & metadata extraction (APK binary XML + AAB via bundletool) |
| `tools/config.rs` | Tool config persistence (`tools_config.json`) |
| `tools/download.rs` | Download & extract ADB, bundletool, Java |
| `tools/paths.rs` | Platform-specific tool paths |
| `tools/profiles.rs` | Signing profile presets (AES-256-GCM encrypted) |
| `tools/recent.rs` | Recent files tracking |
| `tools/status.rs` | Tool status & staleness checks |

### Operation State Machine

Operations use a discriminated union `OperationState` (`idle | installing | extracting`), not boolean flags. Each operation gets a unique cancellation token.

## CI / CD Pipeline

Triggered by **version tags** (`v*`) pushed to `main`:

1. **Build jobs** (parallel): macOS ARM64, macOS x64, Windows x64, Linux x64
2. **Create Release draft** with all platform artifacts
3. **`update-updater-json` job** — runs `generate-updater-json.mjs` to fetch `.sig` files, write `updater.json`, and commit it to `main`

The `updater.json` is served via `https://raw.githubusercontent.com/<repo>/main/updater.json` for the auto-updater.

**Required GitHub Secrets:**
- `TAURI_SIGNING_PRIVATE_KEY` — Ed25519 key for update signing
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — (optional)
- Apple signing secrets (optional, for notarized macOS builds)

## Testing

### Frontend (Vitest + React Testing Library)

- Tests in `src/__tests__/`
- One test file per component/hook (e.g., `DeviceSection.test.tsx`, `useWirelessAdb.test.ts`)
- Setup file: `src/__tests__/setup.ts`
- Mocks Tauri's `invoke` and `event` modules

### Rust (cargo test)

- Tests in each Rust source file (unit tests with `#[cfg(test)]`)
- Run: `npm run test:rust` or `cd src-tauri && cargo test`

### Combined

- `npm run test:all` — runs both frontend and Rust tests
- The release script runs `test:all` before proceeding (can skip with `--skip-tests`)

## Common Patterns

### Adding a feature end-to-end

1. **Rust command** → `src-tauri/src/<module>.rs`, register in `lib.rs`
2. **TypeScript wrapper** → `src/api.ts`
3. **TypeScript types** → `src/types.ts` (if new structs)
4. **Hook** → `src/hooks/use<Feature>.ts` or extend existing hook
5. **Component** → `src/components/<Feature>.tsx` or extend existing
6. **Wire up** → `App.tsx`
7. **Tests** → `src/__tests__/<Feature>.test.tsx`
8. **Styles** → `src/App.css`
9. **Docs** → update `architecture.md`, `feature-analysis.md`, `CHANGES.md`

### Commit workflow (during development)

```bash
# Use prefixed commits
git commit -m "add: new feature description"
git commit -m "fix: bug description"
git commit -m "update: refactoring description"
```

### Windows-specific: Command execution

All subprocess commands use `CREATE_NO_WINDOW` on Windows (via `cmd.rs` helpers) to prevent blank console windows from appearing and stealing focus.
